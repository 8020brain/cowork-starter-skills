# Post Writing Guide

Guidelines for writing Circle, LinkedIn, and blog posts. These rules inform the post-evaluator scoring and should be customised to match your own voice and brand.

## File Naming Convention

**Format:** `[platform]-YYYYMMDD-descriptive-title.md`

**Examples:**
- `circle-20251011-self-improving-agency.md`
- `linkedin-20251011-self-improving-agency.md`
- `email-20251015-weekly-update.md`

**Platform prefixes:**
- `circle-` for Circle community posts
- `linkedin-` for LinkedIn posts
- `email-` for email content

## Writing Style Rules

### Always Write to One Person
- Write as if speaking to a single reader
- Use "you" not "you all". Never "some of you"

### Avoid Common Patterns
- Never use: "It's not about X, it's about Y" structure
- Never use em dashes (--)
- Use regular dashes (-) or commas instead
- NEVER use patterns like "But here's the problem:"
- NEVER use "Here's what most people miss" or variations ("Here's the thing most people don't realize")
- NEVER use "Here's why that matters" or "Here's what changed"
- NEVER use generic "most people" framing - be specific about WHO
- NEVER write in a way that sounds like AI-generated marketing copy
- Avoid clickbait sentence starters that promise a reveal ("The secret is...", "What nobody tells you...")

### Voice and Tone
- Direct and conversational
- Short sentences work
- No unnecessary superlatives
- Practical over theoretical. Make it concrete not abstract.
- Use natural contractions (I'm, can't, you're)
- Write how you'd actually talk, not how you'd write an essay

### Openings and Hooks
- Lead with the problem or insight, not preamble
- Rhetorical questions that engage: "Sounds fair, right?", "Want to know why?"
- NO formal transitions like "Let's unravel", "Here's to", "Today we dive into"
- The first 2 lines must earn the rest of the post

### What to Avoid (AI-isms)
These patterns scream "written by AI" - avoid them completely:
- "Then I hit a wall" - dramatic narrative transitions
- "Here's what changed everything" - overhyped revelations
- "The irony?" - rhetorical questions as transitions
- "But here's the thing:" - formulaic setup lines
- "Let me explain" / "Let me break this down"
- "Game-changer" / "changed the game"
- "Here's why this matters"
- "The secret is..." / "The key is..."
- Any sentence that sounds like a YouTube video title

## Platform-Specific Formatting

### Circle Posts

Circle has specific formatting requirements that differ from standard Markdown.

**Headings:**
- **DON'T** use Markdown heading syntax (`#`, `##`, etc.)
- **DO** use bold text with extra spacing: `**Heading Text**`
- Add TWO blank lines before each heading
- Add ONE blank line after the heading before content

**Bullet Points:**
- Use standard Markdown bullet syntax (`-` or `*`)
- Don't add extra spaces before bullet points
- Keep consistent spacing between bullet items
- Don't nest bullets too deeply (2 levels max recommended)

**Spacing Between Sections:**
- Two blank lines before headings
- One blank line after headings
- One blank line between paragraphs
- One blank line before and after bullet lists

**Example Circle Structure:**
```
**Main Topic**

Introduction paragraph goes here.

Some more text.


**First Section**

Content for this section.

- Bullet point one
- Bullet point two
- Bullet point three


**Second Section**

More content here.
```

**Circle Quick Checklist:**
- All headings use `**bold**` instead of `#`
- Two blank lines before each heading
- One blank line after each heading
- No extra spaces before bullet points
- Consistent spacing throughout
- Simple, clean formatting only

### LinkedIn Posts

**Formatting:**
- Use bold Unicode headers or regular bold for section titles
- Use separators for visual breaks between sections
- Use arrows for bullet points (->)
- Break up text with white space
- Keep to ~50 lines of text. 80 max.

**Style:**
- No hashtags
- Limited if any emojis (0-2 max)
- Direct and punchy
- Lead with the insight or result

**What wins on LinkedIn (2025+ algorithm):**
- Attention/dwell time (people slow down and read)
- Comment depth (replies-to-replies)
- Semantic consistency (stay in your lane)
- Practitioner voice (lived specifics, not polished AI-speak)

**High-attention post types:**
1. Contrarian + proof
2. Tactical checklist/SOP
3. Case teardown
4. Framework
5. Before/After

### Blog Posts

**Formatting:**
- Use # for the main title
- Use ## for section headings
- Keep paragraphs short
- Include clear structure: intro, body, conclusion
- Appropriate depth and length for the topic

## Content Guidelines

### Structure
- Lead with the problem or insight
- Be specific with examples
- Show, don't just tell
- End with clear next step

### Questions
- Ask better questions (don't be vague)
- Questions should be concrete, not abstract
- Use questions that readers actually ask themselves

### Examples
- Use real scenarios wherever possible
- Be specific about outcomes
- Reference actual tools/systems when relevant

## Common Mistakes to Avoid

**Writing:**
- Writing to a crowd instead of one person
- Using "It's not X, it's Y" construction
- Em dashes instead of regular dashes (-)
- Generic advice without concrete examples
- Overly formal or corporate tone
- Too many emojis

**Circle Formatting:**
- Using `#` headings instead of `**bold**`
- Forgetting extra blank line before headings
- Adding spaces before bullet points
- Using complex Markdown (Circle has limited support)

## Summary Checklist

When writing a post, verify:
- [ ] Direct "you" language (one person, not a crowd)
- [ ] Conversational openings (no "Let's dive into")
- [ ] Concrete examples (not vague benefits)
- [ ] Natural voice (contractions, casual tone where appropriate)
- [ ] No filler words or repetitions
- [ ] Platform-appropriate formatting
- [ ] Clear next step or takeaway
- [ ] Practical focus (not purely theoretical)
- [ ] No banned AI-ism phrases
